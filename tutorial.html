Chapter 1 / 下载实战

下载实战

方法 1: Python boto3

import boto3
from botocore import UNSIGNED
from botocore.config import Config

s3 = boto3.client('s3',
  endpoint_url='https://object.empirical.network',
  config=Config(signature_version=UNSIGNED))

# 列出某天文件
resp = s3.list_objects_v2(
  Bucket='openintel',
  Prefix='dnsgraph/ch/warehouse/year=2026/month=04/day=13/')

for obj in resp['Contents']:
    s3.download_file('openintel', obj['Key'],
                     obj['Key'].split('/')[-1])

方法 2: wget / curl

# 直接 HTTP 下载 (需知道文件名)
wget https://object.empirical.network/\
  openintel/dnsgraph/ch/warehouse/\
  year=2026/month=04/day=13/part-0.parquet

方法 3: DuckDB 直接分析

-- 无需下载！直接从 S3 查询
SELECT query_type, count(*)
FROM read_parquet('s3://openintel/
  dnsgraph/ch/warehouse/
  year=2026/month=04/day=13/*.parquet')
GROUP BY query_type
ORDER BY count(*) DESC
LIMIT 10;

Chapter 2 / Parquet

Parquet 格式特性

|||

列式存储

只读取查询需要的列，跳过其余 90+ 列。分析场景下比 CSV 快 10-100x。

zip

高效压缩

同列数据类型一致，压缩率极高。11.2 GB Parquet vs 估算 ~80 GB CSV。

{ }

内嵌 Schema

每个文件自带列名/类型定义，无需额外元数据文件。Schema 可演进。

-- 查看 Parquet Schema
DESCRIBE SELECT * FROM read_parquet('data/ch/*.parquet') LIMIT 0;

-- 只读取需要的列 (列裁剪)
SELECT query_name, ip4_address, country
FROM read_parquet('data/ch/*.parquet')
WHERE query_type = 'A' AND country = 'CH';
-- DuckDB 自动做谓词下推 + 列裁剪

数据质量评估

GeoIP 命中率

~85%

稀疏列注意事项

ip4_address 仅在 query_type='A' 时有值 (~17.9%)
mx_address 仅在 query_type='MX' 时有值 (~9.7%)
DNSSEC 列 (ds_*, dnskey_*) 大量为 NULL

分析建议

Chapter 3 / 查询基础

DuckDB + Parquet 查询基础

-- 单文件查询
SELECT * FROM read_parquet('data/ch.parquet')
LIMIT 10;

-- Glob 多文件查询
SELECT count(*)
FROM read_parquet('data/*.parquet');

-- 带 filename 列
SELECT filename, count(*)
FROM read_parquet('data/*.parquet',
     filename=true)
GROUP BY filename;
-- 创建视图方便反复查询
CREATE VIEW dns AS
SELECT * FROM read_parquet(
  'data/*.parquet',
  hive_partitioning=true);

-- 然后像普通表一样查询
SELECT query_type, count(*) as cnt
FROM dns
GROUP BY query_type
ORDER BY cnt DESC;

-- 导出结果
COPY (SELECT ...) TO 'output.csv';

算法迁移趋势

ECDSA P-256 已成为事实标准 (61.1%)，密钥仅 64 字节
RSA/SHA-256 仍占 37.8%，多为早期部署的域名
Ed25519 虽然性能最优，但验证器支持率仍不足
RSA/SHA-1 (算法 5) 已几乎消失 (< 0.4%)

实践建议

新部署 DNSSEC 推荐使用 ECDSA P-256 (算法 13)：密钥小、签名快、DNS 包不易超限。Ed25519 待生态成熟后再迁移。

什么是 CAA?

CAA 记录指定哪些证书颁发机构 (CA) 被授权为该域名签发 TLS 证书。CA 在签发前必须检查 CAA 记录。

为什么部署率这么低?

非强制：不配置 = 允许所有 CA 签发
感知不足：多数域名管理者不知道 CAA
Let's Encrypt 占 69% 体现免费 CA 的统治地位

SELECT caa_value, count(*) AS cnt
FROM dns
WHERE caa_tag = 'issue'
GROUP BY caa_value
ORDER BY cnt DESC LIMIT 10;

戏剧性的差距

ccTLD 的 DNSSEC 部署率是 TopList 的 5-15 倍
.se 达到 21.9%，而 Tranco 仅 ~2%
ccTLD 注册管理机构提供免费 DNSSEC 签名服务

原因分析

欧洲 ccTLD 注册管理机构（如瑞典 IIS、荷兰 SIDN）主动推广 DNSSEC，提供一键启用。而全球性 gTLD (.com/.net) 缺乏类似激励。

RTT 百分位分析 SQL

SELECT
  percentile_cont(0.50) WITHIN GROUP
    (ORDER BY rtt) AS p50,
  percentile_cont(0.95) WITHIN GROUP
    (ORDER BY rtt) AS p95,
  percentile_cont(0.99) WITHIN GROUP
    (ORDER BY rtt) AS p99
FROM dns
WHERE rtt IS NOT NULL;

RTT 异常阈值

< 10ms: 本地/CDN 边缘节点
10-50ms: 正常跨区域延迟
50-200ms: 跨洲际或轻度拥塞
> 500ms: 可能的 DNS 劫持/超时重试

SELECT day,
  count(DISTINCT query_name) FILTER(
    WHERE ds_key_tag IS NOT NULL)
  * 100.0 / count(DISTINCT query_name)
  AS dnssec_pct
FROM read_parquet('data/ch_*/*.parquet',
  hive_partitioning=true)
GROUP BY day ORDER BY day;
